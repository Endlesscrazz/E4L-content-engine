## 1. Grounding (grounding_pass)

Every health claim, product claim, or performance claim in the draft must be
traceable to one of the provided cited source chunks. A claim is grounded if:
  - A cited chunk directly states it (Type A), or
  - A cited chunk logically implies it without adding new facts (Type B).

A claim is NOT grounded if it cannot be traced to any provided chunk, even if
it sounds plausible or generally true in the wellness space.

Set grounding_pass = false if ANY ungrounded claim is present.

## 2. Claim Taxonomy (taxonomy_issues)

Classify every meaningful claim in the draft:

  Type A — Direct paraphrase of source. Citation required. → PASS
  Type B — Logically implied by source but not stated. Citation required. → PASS
  Type C — General knowledge, not E4L-specific. No citation required. → PASS
  Type D — Novel claim not traceable to any cited chunk. → HARD FAIL

Populate taxonomy_issues with the exact text of every Type-D claim found.
Leave empty if all claims are A, B, or C.

## 3. Certainty Inflation (certainty_inflation_issues)

The source corpus uses hedged, evidence-appropriate language. The draft must not
escalate certainty beyond what the source supports.

FAIL patterns — source says X / draft claims Y:
  "may support energy"         → "restores your energy"       (modal → guarantee)
  "some clients report"        → "proven to improve"          (anecdote → proof)
  "preliminary research suggests" → "science shows"           (preliminary → established)
  "associated with improved wellbeing" → "improves wellbeing" (correlation → causation)
  "many users notice"          → "you will notice"            (many → universal)
  "supports healthy cellular function" → "heals cells"        (support → direct action)

PASS patterns (hedging preserved):
  Source: "may support energy"     → Draft: "may support your energy levels"
  Source: "research suggests"      → Draft: "emerging research suggests"
  Source: "some clients report"    → Draft: "many clients report noticing"

Populate certainty_inflation_issues with the exact draft text of each inflated
claim. Leave empty if hedging is faithfully preserved throughout.

## 4. Voice (voice_pass)

E4L content follows Harry Massey's distinctive structural voice. Use the
provided voice anchor examples as your reference. Key patterns:

  ✓ Opens with a personal narrative, a physics observation, or a direct question
  ✓ Introduces concepts through everyday analogies (tuning forks, water states,
    batteries, signal vs noise, gel vs liquid water)
  ✓ Names specific scientists by name when citing research (Pollack, Popp,
    Lipton, Sheldrake, Szent-Györgyi) — not generic "researchers say"
  ✓ Lands the key takeaway in a short, punchy concluding line
  ✓ Avoids generic wellness marketing clichés ("transform your life",
    "unlock your potential", "game-changer", "revolutionary")

Set voice_pass = false only if the draft reads as generic wellness content with
no trace of Harry Massey's style. Minor deviations from one pattern are
acceptable if the overall voice is authentic to the provided anchors.

## 5. Tone (tone_pass)

Check whether the draft's tone register matches the ContentBrief tone field:

  conversational — plain language, first-person, accessible metaphors, informal
  educational    — structured explanation, cause-and-effect, moderately formal
  clinical       — precise terminology, mechanism references, study citations
  inspirational  — emotionally resonant, narrative-driven, forward-looking

Set tone_pass = false only if there is a clear mismatch (e.g., brief says
"conversational" but draft is dense with clinical jargon).
